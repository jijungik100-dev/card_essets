# 🏷️ 카드게임 효과 시스템 정의서 (v2)

> 끌어당김, 위치교환, 은폐, 정보공개 삭제. 감전/혼란 통합, 약화/둔화 분리 반영.
> 하단에 코드 구조화용 에이전트 프롬프트 포함.

---

## 1. 상태이상 (Debuff) — 10종

적에게 부여하는 지속 효과. 일부 카드는 특정 상태이상에 반응해 추가 데미지를 줌.

| 이모지 | ID | 이름 | 효과 | 중첩 | 기본 지속 |
|--------|----|------|------|------|-----------|
| 🩸 | `BLEED` | 출혈 | 턴 종료 시 중첩당 1 도트 데미지 | ✅ 중첩 가능 | 2턴 |
| ☠️ | `POISON` | 중독 | 매 턴 시작 시 중첩당 1 도트 데미지 | ✅ 중첩 가능 | 3턴 |
| 🧊 | `FREEZE` | 빙결 | 이동·공격 모두 불가 (완전 행동 봉쇄) | ❌ 중첩 불가 (지속만 갱신) | 1턴 |
| 🔥 | `BURN` | 화상 | 매 턴 중첩당 도트 데미지. 화상 폭발 카드로 즉발 전환 가능 | ✅ 중첩 가능 | 2턴 |
| 😱 | `FEAR` | 공포 | 적이 자발적으로 1칸 후퇴. 함정 콤보 핵심 | ❌ 중첩 불가 (지속만 갱신) | 1턴 |
| ⚡ | `SHOCK` | 감전 | 행동 시 50% 확률로 실패 (이동·공격 모두 해당) | ❌ 중첩 불가 (지속만 갱신) | 1턴 |
| 🔗 | `BIND` | 속박 | 이동 불가 (공격은 가능). 넉백 면역 | ❌ 중첩 불가 (지속만 갱신) | 2턴 |
| 💀 | `MARK` | 표식 | 다음에 받는 데미지 +3. 피격 시 소멸 (일회성) | ❌ 일회성 소모 | 피격 시 소멸 |
| 😵 | `WEAKEN` | 약화 | 공격력 -N (카드에 명시된 수치만큼) | ❌ 중첩 불가 (강한 값 우선) | 2턴 |
| 🐌 | `SLOW` | 둔화 | 이동력 -1. 이동은 가능하나 느려짐 | ❌ 중첩 불가 (지속만 갱신) | 2턴 |

---

## 2. 버프 (Buff) — 5종

아군 또는 플레이어에게 부여하는 긍정 효과.

| 이모지 | ID | 이름 | 효과 | 지속 |
|--------|----|------|------|------|
| 🛡️ | `DEFENSE` | 방어 | 플레이어가 받는 피해를 N만큼 감소 | 해당 턴 |
| 💉 | `LIFESTEAL` | 흡혈 | 가한 데미지의 N%를 플레이어 HP로 회복 | 즉발 |
| ❤️ | `HEAL` | 회복 | 플레이어 HP를 N만큼 즉시 회복 | 즉발 |
| 💪 | `ATK_UP` | 공격력 증가 | 이번 턴 아군 카드 데미지 +N | 해당 턴 |
| 🎯 | `ACC_DOWN` | 적 명중률 감소 | 적 전체의 명중률 -N% | 해당 턴 |

---

## 3. 전장 효과 (Field Effect) — 5종

위치 이동, 데미지 전환 등 전장 상태를 직접 조작하는 효과.

| 이모지 | ID | 이름 | 효과 |
|--------|----|------|------|
| 🌀 | `KNOCKBACK` | 넉백 | 적을 뒤로 N칸 강제이동. 이동 경로에 함정 있으면 발동 |
| 🔄 | `REFLECT` | 반사 | 적이 공격 시 자신에게 N 데미지 반사. 지속턴 명시 |
| ☠️💥 | `STATUS_BURST` | 상태이상 폭발 | 대상의 모든 상태이상 즉시 소멸 → 남은 턴 합계 × N 즉발 데미지 |
| 🔥💥 | `BURN_BURST` | 화상 폭발 | 화상 상태 적의 누적 도트 합계 × N 즉발 데미지. 화상 소멸 |
| 🚫 | `IMMUNE` | 피해 무효 | 이번 턴 플레이어가 받는 모든 피해 무효 |

---

## 4. 리소스 효과 (Resource) — 4종

에너지, 카드 드로우, 코스트 등 게임 리소스를 관리하는 효과.

| 이모지 | ID | 이름 | 효과 |
|--------|----|------|------|
| 🃏 | `DRAW` | 추가 드로우 | 덱에서 카드 N장 추가 드로우 |
| ⚡➕ | `ENERGY_GAIN` | 에너지 획득 | 이번 턴 사용 가능 에너지 +N |
| ⚡➖ | `COST_REDUCE` | 코스트 할인 | 지정 카드의 사용 코스트 -N (최소 1) |
| ♻️ | `RECYCLE` | 카드 회수 | 사용된 카드 또는 필드 함정을 패로 복귀 |

---

## 5. 함정 키워드 (Trap) — 3종

함정/설치 카드 전용 키워드.

| 이모지 | ID | 이름 | 효과 |
|--------|----|------|------|
| ⏱️ | `DURATION` | 지속턴 | 함정이 필드에 유지되는 턴 수 |
| 📍 | `DEPLOY` | 설치 | 지정 레인의 특정 칸에 배치 |
| 💥 | `TRIGGER` | 진입 발동 | 적이 해당 칸 진입 시 자동 발동 |

---

| 대분류 | 종류 수 | 목록 |

| 버프 | **5종** | 방어, 흡혈, 회복, 공격력 증가, 명중률 감소 |

# 🤖 에이전트용 코드 구조화 프롬프트

아래 프롬프트를 코딩 에이전트에게 전달하면, 위 효과 시스템을 게임 코드에 구현할 수 있습니다.

---

```
당신은 모바일 카드게임의 전투 시스템을 구현하는 게임 개발자입니다.
아래에 정의된 효과 시스템을 코드 레벨로 구조화해주세요.

## 요구사항

### 1. Enum 정의
모든 효과를 카테고리별 Enum으로 정의해주세요.

- **StatusEffect** (상태이상 10종):
  BLEED, POISON, FREEZE, BURN, FEAR, SHOCK, BIND, MARK, WEAKEN, SLOW

- **BuffEffect** (버프 5종):
  DEFENSE, LIFESTEAL, HEAL, ATK_UP, ACC_DOWN

- **FieldEffect** (전장 효과 5종):
  KNOCKBACK, REFLECT, STATUS_BURST, BURN_BURST, IMMUNE

- **ResourceEffect** (리소스 4종):
  DRAW, ENERGY_GAIN, COST_REDUCE, RECYCLE

- **TrapKeyword** (함정 3종):
  DURATION, DEPLOY, TRIGGER

### 2. 효과 데이터 구조 (EffectDefinition)
각 효과는 다음 속성을 가집니다:
- id: string (예: "BLEED")
- name: string (예: "출혈")
- emoji: string (예: "🩸")
- category: "debuff" | "buff" | "field" | "resource" | "trap"
- stackable: boolean (중첩 가능 여부)
- defaultDuration: number | null (기본 지속턴, null이면 즉발 또는 조건부 소멸)
- dotPerStack: number | null (중첩당 도트 데미지, 없으면 null)
- description: string (효과 설명)

### 3. 상태이상 동작 규칙

**도트 계열 (BLEED, POISON, BURN)**:
- 중첩 가능. 새로 부여 시 기존 중첩에 추가.
- 지속턴은 가장 긴 값으로 갱신 (개별 타이머가 아닌 단일 타이머).
- BLEED: 턴 종료 시 발동
- POISON: 턴 시작 시 발동
- BURN: 턴 시작 시 발동. BURN_BURST 카드 사용 시 (누적 도트 합계 × 배율) 즉발 데미지 후 화상 제거.

**행동 제한 계열 (FREEZE, SHOCK, BIND, SLOW)**:
- 중첩 불가. 새로 부여 시 지속턴만 갱신 (더 긴 값 적용).
- FREEZE: 이동·공격 모두 불가.
- SHOCK: 행동 시도 시 50% 확률로 실패 (이동·공격 각각 판정).
- BIND: 이동 불가, 공격 가능. 넉백(KNOCKBACK) 효과 면역.
- SLOW: 이동력 -1 (이동은 가능).

**특수 계열 (FEAR, MARK, WEAKEN)**:
- FEAR: 적의 턴 시작 시 자동으로 1칸 후퇴. 후퇴 칸에 함정 있으면 즉시 발동.
- MARK: 일회성. 대상이 다음 피격 시 데미지 +3, 피격 후 소멸. 지속턴 없음 (피격 시 제거).
- WEAKEN: 공격력 -N. 중첩 불가, 더 강한 값이 우선 적용.

### 4. 화상 폭발 / 상태이상 폭발 로직

**BURN_BURST (화상 폭발)**:
- 화상 상태인 모든 적(또는 지정 적)에게 발동.
- 즉발 데미지 = 해당 적의 (현재 화상 중첩 수 × 남은 턴 수의 도트 합계) × 카드 명시 배율.
- 발동 후 화상 상태 제거.

**STATUS_BURST (상태이상 폭발)**:
- 대상 적 1명에게 발동.
- 대상의 모든 상태이상을 즉시 제거.
- 즉발 데미지 = 제거된 모든 상태이상의 남은 지속턴 합계 × 카드 명시 배율.
- 일회성 상태(MARK 등)는 남은 턴 1로 계산.

### 5. 넉백 + 함정 연동

- KNOCKBACK 발동 시 적이 N칸 뒤로 이동.
- 이동 경로의 각 칸에 함정(TRIGGER 타입)이 있으면, 해당 칸 도착 시 함정 즉시 발동.
- BIND 상태의 적은 넉백 면역.
- FEAR에 의한 자발적 후퇴도 함정 발동 대상.

### 6. 카드 데이터 구조 (CardDefinition)
각 카드는 다음 속성을 가집니다:
- id: string (예: "A01")
- name: string (예: "배달 드론 36호")
- description: string (캐릭터 설명)
- category: "physical" | "magic" | "support" | "trap" | "utility"
- cost: number (에너지 코스트)
- effects: Effect[] (아래 구조의 배열)

각 Effect는:
- type: "damage" | "status" | "buff" | "field" | "resource"
- target: "front_enemy" | "lane_enemy" | "all_enemies" | "random_enemies" | "marked_enemy" | "self" | "player" | "lane_deploy"
- targetCount: number | null (대상 수, null이면 전체)
- value: number (데미지 수치, 회복량, 버프 수치 등)
- statusEffect: StatusEffect | null (부여할 상태이상)
- stackCount: number | null (중첩 수)
- duration: number | null (지속턴)
- condition: string | null (조건부 효과 설명, 예: "target_hp_below_35%")
- bonusValue: number | null (조건 충족 시 추가 수치)

### 7. 출력 형식
- TypeScript로 작성해주세요.
- 모든 Enum, 인터페이스, 타입을 먼저 정의하고,
- 효과 정의 테이블(EffectDefinition[])과 카드 데이터(CardDefinition[])를 분리해주세요.
- 상태이상 처리 로직은 BattleEffectProcessor 클래스로 구현하고,
  각 상태이상의 발동 타이밍(턴 시작/턴 종료/피격 시/행동 시)별로 메서드를 분리해주세요.
- 넉백→함정 연동, 화상 폭발, 상태이상 폭발 로직을 별도 메서드로 구현해주세요.
```
